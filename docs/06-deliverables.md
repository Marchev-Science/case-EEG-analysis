## What you submit

1. **Report** at `reports/<your-name>.md` (or `.pdf`) — see [`../reports/TEMPLATE.md`](../reports/TEMPLATE.md).
2. **Code** under `code/` — runnable, with a `requirements.txt` you actually used.
3. **Raw data** under `data/raw/` *or* a link (Drive/OSF/Zenodo) if too large.
4. **Pre-registration** (a short markdown file in your report folder, dated
   *before* you started analysis).
5. **Filled protocol** — copy of the relevant `protocols/0X-*.md` adapted to
   what you actually did.

## Suggested grading rubric

| Dimension                      | Weight |
|--------------------------------|--------|
| Question clarity & motivation  | 10 %   |
| Experimental design rigor      | 20 %   |
| Data collection quality        | 15 %   |
| Analysis correctness           | 25 %   |
| Reproducibility (code + data)  | 15 %   |
| Honest discussion of limits    | 15 %   |

Grading is biased *toward* honest negative results and methodological care
and *against* over-claiming on small samples.

## Submission

Open a pull request from your fork, or push to a branch named
`student/<your-name>`. Include all five items above.
